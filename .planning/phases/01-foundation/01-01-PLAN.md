```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <!-- Required: App Sandbox (D-04) -->
    <key>com.apple.security.app-sandbox</key>
    <true/>

    <!-- Required: Bluetooth access for IOBluetooth (D-02, INF-01) -->
    <key>com.apple.security.device.bluetooth</key>
    <true/>

    <!-- CloudKit — configured now, cannot test until Developer Account exists (D-01, D-03, INF-02) -->
    <key>com.apple.developer.icloud-containers-environment</key>
    <string>Production</string>
    <key>com.apple.developer.icloud-container-identifiers</key>
    <array>
        <string>iCloud.$(PRODUCT_BUNDLE_IDENTIFIER)</string>
    </array>
    <key>com.apple.developer.ubiquity-kvstore-identifier</key>
    <string>$(TeamIdentifierPrefix)$(CFBundleIdentifier)</string>
</dict>
</plist>
```

1. Selecting the SyncBuds target
2. Going to Signing & Capabilities tab
3. Setting the Code Signing Entitlements build setting to "SyncBuds/SyncBuds.entitlements" for the macOS destination

If modifying project.pbxproj directly is required, set CODE_SIGN_ENTITLEMENTS = SyncBuds/SyncBuds.entitlements in the macOS build configuration section.

IMPORTANT: The CloudKit entitlements are stubs only — they will not pass code signing until an Apple Developer Account exists (D-01). That is expected and correct for this phase.
