  </action>
  <what-built>
    Bidirectional headphone switching state machine (SwitchCoordinator) wired to BluetoothManager and MultipeerService. Switch button in ContentView triggers the coordinator. System notifications on completion. Cooldown window suppresses auto-reconnect.
  </what-built>
  <how-to-verify>
    Prerequisites:
    - Build and run the macOS app (Cmd+R, target: My Mac)
    - Build and run the iOS app (target: your iPhone)
    - Both devices on the same Wi-Fi network
    - Your Bluetooth headphone connected to Mac

    Step 1 — Confirm peer connection:
    Both apps should show "Connected to [device name]" in the Peer Status section.
    Expected: green dot, peer name visible.

    Step 2 — Mac→iPhone switch:
    On Mac app, tap "Switch Headphone".
    Expected within 5 seconds:
    - Mac app label changes to "Switching..." then "Cooldown..."
    - Headphone disappears from Mac's System Settings > Bluetooth > Connected
    - Mac shows system notification: "Headphone Switched — Now available for iPhone"
    - iOS app shows system notification (when Mac sends switchRequest)
    After ~10 seconds: Mac button returns to "Switch Headphone" (cooldown ended).
    Optional: Connect headphone on iPhone to confirm it pairs successfully.

    Step 3 — iPhone→Mac switch:
    On iPhone app, tap "Switch Headphone".
    Expected within 15 seconds:
    - iOS app label changes to "Switching..."
    - Mac receives switchRequest and calls openConnection()
    - Headphone appears on Mac's System Settings > Bluetooth > Connected
    - Mac sends .status("connected") to iOS
    - iOS shows system notification: "Headphone Switched — Now connected to Mac"
    - iOS button returns to "Switch Headphone"

    Step 4 — Race condition guard (SW-05):
    On Mac, tap "Switch Headphone" twice rapidly.
    Expected: Only one switch initiates. Second tap is silently ignored (button disabled or console shows "ignoring duplicate request").

    Step 5 — Cooldown suppression (SW-05):
    After Mac→iPhone switch, observe that the headphone does NOT immediately reconnect to Mac.
    Expected: During the ~10-second cooldown window, if headphone tries to auto-reconnect, it is immediately disconnected again. Console should show "[BluetoothManager] Suppressing auto-reconnect during cooldown: [name]" if auto-reconnect fires.
  </how-to-verify>
  <resume-signal>
    Type "approved" if both directions work and notifications appear.
    Or describe any failure: which step failed, what happened instead, any error messages from Xcode console.
  </resume-signal>
