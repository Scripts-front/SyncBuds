Public interface:
- `func start()` — calls `advertiser.startAdvertisingPeer()` and `browser.startBrowsingForPeers()`

- `func send(_ signal: SyncSignal) throws` — guards on `session.connectedPeers.isEmpty`; encodes with `JSONEncoder()`; calls `session.send(data, toPeers: session.connectedPeers, with: .reliable)`

MCNearbyServiceBrowserDelegate:
- `browser(_:foundPeer:withDiscoveryInfo:)` — only invite if `session.connectedPeers.isEmpty` (Pitfall 6 fix); call `browser.invitePeer(peerID, to: session, withContext: nil, timeout: 10)`
- `browser(_:lostPeer:)` — log only, no action needed

MCNearbyServiceAdvertiserDelegate:
- `advertiser(_:didReceiveInvitationFromPeer:withContext:invitationHandler:)` — call `invitationHandler(true, session)` immediately

MCSessionDelegate (all callbacks dispatch UI mutations to DispatchQueue.main.async — Pitfall 4):
- `session(_:peer:didChange:)`:
  - `.connected`: set `isConnectedToPeer = true`, `connectedPeerName = peerID.displayName`, call private `startStatusTimer()`; log with print
  - `.connecting`: log only
  - `.notConnected`: set `isConnectedToPeer = false`, `connectedPeerName = nil`, `peerBluetoothStatus = "unknown"`, call private `stopStatusTimer()`; then call `browser.startBrowsingForPeers()` to re-discover when peer returns; log with print
  - `@unknown default`: break
- `session(_:didReceive:fromPeer:)`:
  - Decode with `JSONDecoder()` — if decode fails, return silently
  - Check `signal.isFresh` — if false, return silently (log rejection)
  - `DispatchQueue.main.async`: call private `handleReceivedSignal(signal)`
- Three required stub methods (stream, startReceivingResource, finishReceivingResource) — empty bodies

Private helpers:
- `private var statusTimer: Timer?`
- `private func startStatusTimer()` — `Timer.scheduledTimer(withTimeInterval: 5.0, repeats: true)` that sends a `.status` signal with current `localBluetoothStatus`. Guard: `isConnectedToPeer` must be true inside timer callback.
- `private func stopStatusTimer()` — invalidates + nils the timer
- `private func handleReceivedSignal(_ signal: SyncSignal)` — updates `peerBluetoothStatus = signal.bluetoothStatus`; log with print. Future phases will add `.switchRequest` handling here; add a switch with a TODO comment for that case.
